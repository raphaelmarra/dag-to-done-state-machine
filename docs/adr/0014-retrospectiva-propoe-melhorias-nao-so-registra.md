# 0014 — Retrospectiva propõe melhorias, não só registra

- Status: accepted
- Data: 2026-06-26
- Origem: migrado de DECISOES.md (D014)

## Contexto e Problema
O gate-ledger registra bugs mas ninguém propõe correção estrutural do pipeline.

## Decisão
A Retrospectiva não só registra lições — o agente propõe explicitamente melhorias: critérios a mudar, lentes a adicionar, etapas a ajustar.

## Motivo
Um registro sem proposta de ação é um arquivo. Uma proposta de melhoria é o mecanismo de evolução do pipeline. O pipeline precisa aprender com cada ciclo.

## Consequências
As propostas precisam ser encaminhadas (não só registradas). Quem decide se aplica é o humano — o agente propõe, não decide.
